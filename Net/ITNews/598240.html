传微软拟收购创企GitHub 打造对抗亚马逊王牌武器
投递人 itwriter 发布于 2018-06-02 13:18 原文链接 [收藏] « »

　　BI 中文站 6 月 2 日报道

　　微软已经与面向开源及私有软件项目的托管平台 GitHub 就收购后者举行了谈判，GitHub 是软件开发行业的知名初创企业，估值约为 20 亿美元，拥有 2400 万软件开发人员（即用户）。

　　目前还不清楚这些会谈的结果，微软拒绝置评。但在我们等待 GitHub 证实消息时，有一件事已经非常明确，那就是微软收购 GitHub 是完全合理的。

微软首席执行官萨蒂亚·纳德拉（Satya Nadella）

　　如果收购成功，GitHub 技术将被整合到微软产品中，让后者在与亚马逊云计算服务 AWS 的竞争中获得巨大优势。亚马逊 AWS 目前依然在快速增长的云计算市场占据领先地位。

　　GitHub 是一项在线服务，允许开发者托管他们的软件项目。在这个平台上，世界各地的任何人都可以下载这些项目，并提交他们自己的改进。这个功能使 GitHub 成为开源软件开发世界的中心。

　　从表面上看，微软收购 GitHub 的逻辑非常清晰。开发者喜欢 GitHub，而微软需要吸引开发者的关注。微软为开发者提供了一系列工具，包括越来越流行的 Visual Studio Code 软件和开源 .NET Core 编程框架。

　　这些工具的流行为微软云计算服务 Azure 和其他微软产品与服务提供了一个温和但明显有效的渠道：如果你喜欢某个微软产品，你更有可能选择其他的微软产品，特别是如果它们能够无缝集成时。

　　GitHub 只会增强这一策略：开发人员已经非常喜欢 GitHub。事实上，微软 2017 年放弃了自家 Codeplex 项目，它是 GitHub 的竞争对手。当时微软表示，GitHub 的受欢迎程度让微软的努力变得多余和没有必要。

　　通过收购 GitHub，微软将直接接触到数以百万计高度活跃的开发人员。正如 GitHub 和微软本月刚刚宣布其服务之间开始集成一样，我们已经看到它们正朝这个方向迈出一小步。

　　不过，如果微软决定收购 GitHub，它可能还有更深层次的打算。

GitHub 联合创始人兼首席执行官克里斯·万斯特拉斯（Chris Wanstrath）

　　除了托管软件项目，GitHub 还将大量资源投入到该网站的社交网络方面，使得其团队和社区更容易在软件项目上合作。

　　与托管软件不同，自己运行软件项目完全是另一回事。即使在 GitHub 上托管的软件经过了调整，并准备好迎接黄金时代到来，开发人员仍然需要下载并运行它。

　　如果它是个相对简单的工具，目的是在 PC 上运行，这可能很容易。但如果它是一个大型复杂的开源基础设施工具，目的是在大型公共云上运行，如亚马逊、微软或谷歌云平台上，则要困难得多。

　　因此，微软收购 GitHub 获得的好处非常明显。如果它能让微软 Azure 与 GitHub 紧密地集成起来，比如让开发者更容易在云端运行 GitHub 项目，它就能一举两得。开发人员可能更喜欢 GitHub，这也将推动微软 Azure 被更多使用。

　　考虑到微软目前专注于其云计算业务的持续爆炸式增长，GitHub 完全可以融入其中。它将成为微软的王牌武器，可以用来缩小 Azure 与 AWS 之间的差距。

　　假设收购能让微软受益还有另一个更微妙的原因。微软斥资 260 亿美元收购职业社交网站 LinkedIn 时，其首席执行官萨蒂亚·纳德拉(Satya Nadella)曾表示，该公司正大举投资，以确保当前和未来的员工具备在现代经济中取得成功所需的技能。

　　至少在硅谷，雇主要求你提供 GitHub 个人简历而不是传统简历，这种情况并不少见。如果微软试图理解现代技能经济，GitHub 可以提供强大的推动力。
